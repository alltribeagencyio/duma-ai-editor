## PROJECT CONTEXT:
Duma AI is a product photography web application that transforms phone photos into studio-quality images using AI. The app uses:
- Webhooks to trigger n8n workflows for image editing
- Supabase for database and storage bucket
- Paystack for payment processing
- Current max upload: 10 images per batch
- Existing features: basic image upload/editing, history page with product-based organization, search by SKU/name/date

## FEATURE #1: IMAGE RE-EDITING/UPSCALING SYSTEM

### Description:
After users receive their initial edited images, they should be able to select any image they like and re-edit it with different prompts for iterative refinement.

### Requirements:
1. **Image Selection Interface:**
   - Add "Re-edit" or "Upscale" button/action on each processed image
   - Allow users to select an image from their results or history page
   - Display the selected image clearly before re-editing

2. **Prompt Selection for Re-editing:**
   - Present three prompt options when re-editing:
     * Custom prompts (user-written text input)
     * Brand prompts (saved brand-specific prompts from onboarding)
     * App preset prompts (default system presets)
   - UI should clearly distinguish between the three prompt types
   - Allow users to preview/modify prompts before submission

3. **Re-editing Process:**
   - Trigger the same webhook → n8n workflow for processing
   - Track that this is a re-edit (not a new upload) in the database
   - Deduct 1 credit per re-edit from user's credit balance
   - Show processing status/loading indicator
   - Notify user when re-edit is complete (WhatsApp if enabled)

4. **Version Tracking:**
   - Link re-edited images to their original source image
   - Store relationship: Original → Re-edit v1 → Re-edit v2, etc.
   - Save metadata: which prompt was used, timestamp, credits consumed

5. **Integration Points:**
   - Must work with existing webhook/n8n/Supabase architecture
   - Add necessary database fields to track image versions and relationships
   - Ensure credit system properly deducts for re-edits
   - Update history page to display re-edited versions (see Feature #8)

## FEATURE #2: BRAND PROMPTS VIA ONBOARDING

### Description:
During user onboarding, create custom brand prompts based on their brand requirements to ensure consistent, on-brand results across all their product images.

### Requirements:
1. **Onboarding Flow Integration:**
   - Add brand prompt creation step in onboarding wizard
   - This step should only appear if user chooses to pay the setup fee
   - Create an intuitive interface for brand prompt configuration

2. **Brand Prompt Creation Interface:**
   - **Brand Information Gathering:**
     * Brand name
     * Industry/product category
     * Brand aesthetic/style preferences (dropdown + text input)
     * Color preferences
     * Background preferences (solid color, gradient, studio, outdoor, etc.)
     * Lighting preferences (bright, moody, natural, etc.)
     * Any specific requirements or restrictions
   
   - **Prompt Templates:**
     * Provide industry-specific templates to help users
     * Allow users to customize templates
     * Preview sample results if possible

3. **Prompt Storage:**
   - Store brand prompts in centralized Supabase prompt database
   - Link brand prompts to user account
   - Allow multiple brand prompts per user (limits: 10 for Starter, 30 for Pro, 100+ for Enterprise)
   - Store metadata: creation date, last used, usage count

4. **Practice Credits:**
   - Upon setup fee payment, allocate practice credits:
     * Starter: 100 free credits
     * Pro: 200 free credits
   - Track practice credits separately from monthly credits
   - Allow users to use practice credits for testing brand prompts OR final production
   - Display practice credit balance clearly in UI

5. **Brand Prompt Management (in app):**
   - Users should be able to view their brand prompts after onboarding
   - Allow editing of existing brand prompts
   - Allow creation of new brand prompts (within tier limits)
   - Show which prompt was used for each image in history

6. **Integration with Image Editing:**
   - Brand prompts should appear as selectable options during image upload
   - Brand prompts should be available during re-editing (Feature #1)
   - Clear labeling to distinguish brand prompts from custom/preset prompts
